## How to Add a Changeset

When you make changes to this package, you should add a changeset to document what has changed. This helps us maintain a clear changelog and version our releases properly.

### Steps to Add a Changeset

1. Run the following command in your terminal:
   ```bash
   npm run changeset
   ```

2. You'll be prompted to:
   - Select the type of change (patch, minor, or major)

   - Write a summary of your changes

3. A new markdown file will be created in `.changeset/` directory with a random name (e.g., `cool-cats-dance.md`)

4. Commit this changeset file along with your code changes

### What Happens Next?

When your PR is merged to the `main` branch, the GitHub Actions workflow will:
1. Detect the changeset
2. Bump the version in `package.json`
3. Update `CHANGELOG.md` with your changes
4. Build the package
5. Publish to npm
6. Create a git tag for the new version

### Important Notes

- **Always add a changeset** when making changes that should be included in the changelog
- Changesets are automatically consumed during the release process
- If you forget to add a changeset, your changes won't be included in the next release
- You can add multiple changesets in a single PR if needed
